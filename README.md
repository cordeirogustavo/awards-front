# Awards Front

A dashboard with stats about the Golden Raspberry Awards movies, plus a list page with filters and pagination.

## Requirements

- Node **24** (this project was built against `v24.15.0`, set in `.nvmrc`). If you use `nvm`, just run:

```bash
nvm use
```

## Getting started

1. Install dependencies:

```bash
npm install
```

2. Copy `.env-example` to `.env` and adjust if needed:

```bash
cp .env-example .env
```

| Variable            | Description                               |
| ------------------- | ----------------------------------------- |
| `VITE_APP_NAME`     | Name shown in the app                     |
| `VITE_API_BASE_URL` | Base URL of the API used by the front-end |

3. Start the dev server:

```bash
npm run dev
```

The app runs at `http://localhost:3000`.

## Scripts

| Command                   | What it does                                            |
| ------------------------- | ------------------------------------------------------- |
| `npm run dev`             | starts the dev server                                   |
| `npm run build`           | builds the app for production                           |
| `npm run preview`         | serves the production build locally                     |
| `npm run test`            | runs the test suite with Vitest                         |
| `npm run lint`            | runs Biome's linter                                     |
| `npm run format`          | formats the code with Biome                             |
| `npm run check`           | runs lint + format in check mode                        |
| `npm run generate:routes` | generates TanStack Router routes from `src/routes`      |
| `npm run generate:api`    | generates the API clients/hooks from the swagger (Kubb) |

## Project structure

```
src/
├── api/generated/        # code generated by Kubb from the API swagger — don't edit by hand
├── client/                # fetch instance used by the generated hooks
├── components/            # shared UI components (data-table, layout, dropdown, etc.)
├── config/                # app config (reads the env variables)
├── context/               # React contexts (e.g. sidebar)
├── hooks/                  # generic hooks (use-debounce, use-mobile)
├── lib/                    # utility functions (cn, etc.)
├── pages/
│   ├── dashboard/          # dashboard page with the 4 panels
│   └── list/               # movies list, with year/winner filters and pagination
├── routes/                 # TanStack Router routes (file-based routing)
├── main.tsx
└── router.tsx
```

Each module (hook, page, etc.) keeps its tests in a `tests/` folder right next to it — e.g. `src/pages/dashboard/tests/dashboard.hooks.test.ts`.

## Stack

- [TanStack Router](https://tanstack.com/router) for file-based routing (`src/routes`)
- [TanStack Query](https://tanstack.com/query) for data fetching, with hooks generated by [Kubb](https://kubb.dev/) from the API swagger
- [TanStack Table](https://tanstack.com/table) for the tables (`src/components/data-table`)
- [Tailwind CSS](https://tailwindcss.com/) for styling
- [Biome](https://biomejs.dev/) for linting and formatting
- [Vitest](https://vitest.dev/) + Testing Library for tests
